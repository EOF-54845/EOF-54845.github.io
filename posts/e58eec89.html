1. 配置java环境

网上有很多教程，这里略过。

2. 什么是动态调试

所谓动态调试，即在程序运行时跟踪执行流程，观察参数 / 局部变量 / 函数返回值，用以推断加密过程与逻辑。

3. 动态调试步骤

1. 获取debug权限

  1. 在AndroidManifest.xml中，添加可调试权限:android:debuggable="true"
  2. 使用XappDebug模块hook对应的软件，项目地址：XappDebug
  3. magisk命令
  4. 刷入MagiskHide Props Config模块

2. 端口转发与开启adb权限

版本号点击七次开启adb权限
以mumu模拟器为例子，开启adb权限后，找到木木模拟器下的shell文件夹，将其路径添加进环境变量的PATH。

在cmd中输入：adb connect 127.0.0.1:16384

3. 下断点

打开jeb，想调哪一行 → 选中代码 → Ctrl + B

4. 开启debug模式

在本课程中，为：adb shell am start -D -n com.zj.wuaipojie/.ui.MainActivity

4. 实际操作

在教程软件中添加获取debug权限语句后，打开软件，点开第四关，随便输入一个值，提示“密钥错误哦，再想想”。打开jeb，拖入安装包，在下方的quick search中搜索“密钥”。

定位后上下翻阅，发现关键函数check()，跳转后仔细阅读。

阅读后发现，check主要有三部分组成，头、尾与中间值。其中头和尾都是固定值，即’flag{‘与’}'。

至于中间值，虽然无法读懂具体的计算过程，但在最后可以发现关键行

其中p1是通过我们之前输入的值，v0是经过计算后得到的密钥，我们只要获取v0，即可破解密码。

v0是运行中的值，我们可以通过调试来获取。在关键行打断点，打开cmd，连接上模拟器，以我在使用的mumu模拟器举例，在设置中打开adb调试，在cmd中输入adb connect 127.0.0.1:16384即可。

输入上述的debug调试指令后，附上emulator-5554，即可开始debug。
其中跳过即下一步，进入即进入这个语句。

进入语句后，一直跳过，直到右侧出现我们的目标变量的值。我的ID为123，算出的值为’AwwKAw=='。

返回软件，输入’flag{AwwKAw==}'，成功解锁。

6. 实际操作

将课件中的日志插桩文件加入安装包，改名为classes2.dex，反编译classes.dex，搜索“密钥”定位到相关方法。找到check方法并跳转，往下翻，定位到关键函数

通过阅读代码可知，这一行会得到正确结果，其中v0存储了答案。所以在此处插桩。

在253行后加入代码invoke-static {p1}, Lcom/mtools/LogUtils;->v(Ljava/lang/Object;)V，退出软件，打开算法助手，打开log捕获，启动软件，输入’flag{123}'后返回算法助手查看日志。

查看所有日志，其中有一个日志的参数值中含有qtfreet(即编写插桩代码文件的作者)，这个日志中的一个参数的参数值与其他的明显不同，为’AwwKAw==',即我们需要的正确答案。

7. 课后作业

安装课后作业软件，打开后，需要输入用户名和16位注册码。输入后，提示用户名或注册码错误，可以通过本节课学习的内容进行破解。

这里只演示log插桩，jeb调试仅在后面获取正确答案时步骤不同，分析方法一样。

将软件拖入jeb，搜索"注册码"，跳转到对应位置。找到关键函数checkSN()，双击跳转。右键解析smali代码并阅读可知，这个check函数会首先判断你的输入是否合法(用户名和注册码不为空且注册码为16位数)，然后开始计算正确验证码。

计算方式是对输入的用户名生成32位的哈希值，然后取所有偶数位。这里假设我们看不懂，直接定位关键语句:

按照程序运行顺序，可以确认return s3.equalsIgnoreCase(sn);是跳出方法的语句，所以s3应该存储了我们希望的答案，s3是由sb转换而来。

回到smali代码，找到关键方法equalsIgnoreCase，上一行既是存储答案的v6，由v5转换而来，和之前的分析一致。找到存储答案的变量后，开始插桩。

将插桩文件拖入并改名，搜索equalsIgnoreCase，在115行后写入插桩代码。保存并安装后，返回算法助手，勾选作业app，lsposed中将算法助手的作用域扩展到作业app，在算法助手中打开日志记录。

翻看日志，找到参数值包含qtfreet的日志，其中第二个参数是一个十六位字符串，回到作业app输入后，成功注册。

8. 总结

本次课学习了jeb动态调试与log插桩，这是两种非常重要的逆向手段，可以帮助我们在不能完全理解被混淆后的代码的情况下，推断出代码的功能，并获取想要的变量值。

个人认为本次的难点还是在于定位关键变量和语句，这需要我们多加实战，积累经验，阅读更多的代码。学习时可以借助ai来辅助理解smali代码，但不能直接将代码丢入ai后只要结果而不自己思考。
